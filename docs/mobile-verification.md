# 모바일 프론트엔드 검증 기록

## 범위

정적 GitHub Pages 배포 구조, 외부 JSON과 HTML 폴백, QR 문자열, 접근 확인 정책,
검색 정렬, 최근 내역 저장 형식을 유지한 상태에서 모바일 레이아웃·상호작용·가독성·성능을 검증했다.

## 자동 검증

- 실행 명령: `npm test`
- Node 버전: GitHub Actions 기준 22
- 검증 대상: 접근 확인, 데이터 최신 상태, 검색 정규화와 정확 일치, 스와이프 판정,
  연속 스캔 상태 이동, QR 작업, 최근 내역, 모바일 CSS, 디자인 토큰, 모션 감소, 렌더링 비용
- CI: 모든 PR과 `main` 푸시에서 `.github/workflows/validate.yml` 실행

## 브라우저 뷰포트

| 뷰포트 | 가로 오버플로 | 본문 크기 | 접근 입력 높이 |
| --- | --- | --- | --- |
| 320×568 | 없음 | 16px | 44px |
| 360×800 | 없음 | 16px | 44px |
| 375×667 | 없음 | 16px | 44px |
| 390×844 | 없음 | 16px | 44px |
| 412×915 | 없음 | 16px | 44px |
| 768×1024 | 없음 | 16px | 44px |
| 1440×900 | 없음 | 16px | 44px |

각 너비에서 `document.documentElement.scrollWidth <= innerWidth`를 확인했다. 320px 잠금 화면은
실제 브라우저 스크린샷으로도 확인했다. `prefers-reduced-motion: reduce` 에뮬레이션에서는 토스트의
CSS 전환이 제거되고 연속 스캔이 모션 생략 분기를 사용하는 것을 확인했다.

## 성능 관련 확인

- 검색용 정규화 문자열과 정확 일치 Map은 데이터 로딩 시 한 번 생성한다.
- 입력 디바운스는 90ms이며 최신 요청 ID만 DOM을 갱신한다.
- 목록은 `DocumentFragment`와 `replaceChildren`으로 일괄 반영한다.
- 선택 변경은 전체 버튼 순회 대신 키 기반 Map으로 이전·현재 버튼만 갱신한다.
- 동일 QR 값과 크기는 다시 생성하지 않는다.
- 반복 목록에는 backdrop blur를 적용하지 않는다.
- 로컬 잠금 화면의 브라우저 측정값은 DOMContentLoaded 약 60ms, 의미 있는 첫 페인트 약 96ms였다.

## 제한사항

- 저장소에 접근 비밀번호 평문이 없으므로 자동 검증에서 잠금 화면 이후의 실제 재고 데이터를 열지 않았다.
  잠금 이후 기능은 순수 함수 테스트, DOM 계약 테스트, 실제 소스 데이터 검증으로 확인했다.
- Lighthouse 패키지를 새 의존성으로 추가하지 않았다. 따라서 점수 목표는 자동 수치로 확정하지 않았으며,
  잠금 화면의 브라우저 타이밍과 구조적 병목 제거 결과만 기록한다.
- Web Share API의 실제 시스템 공유 시트는 브라우저와 OS에 종속되어 구현 분기와 복사 폴백을 테스트했다.
